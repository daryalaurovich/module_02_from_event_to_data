Шаг 1. Определение Первичных Ключей (PK)

Задание: Для каждой из 8 сущностей определите один первичный ключ (PK).

Первичный ключ для сущности categories - category_id
Первичный ключ для сущности cities - city_id
Первичный ключ для сущности countries - country_id
Первичный ключ для сущности customers - customer_id
Первичный ключ для сущности employees - employee_id
Первичный ключ для сущности products - product_id
Первичный ключ для сущности sales - sales_id
Первичный ключ для сущности shops - shop_id

Задание: Обоснуйте свой выбор, исходя из принципа уникальности, на примере сущностей customers и categories.

Для сущности customers поле customer_id является первичным, потому что оно является уникальным для каждого клиента, не повторяется, не зависит от изменяемых данных, однозначно идентифицирует запись в таблице.

Поле category_id для сущности categories выбрано первичным, так как оно гарантирует уникальность каждой категории, названия категорий могут изменяться, не зависит от изменяемых данных, однозначно идентифицирует запись в таблице.
